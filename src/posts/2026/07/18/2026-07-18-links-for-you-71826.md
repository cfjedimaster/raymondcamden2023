## Interactive SVGs and PDFs

As they say, you can take the dev out of PDFs but not the PDFs out of the dev. I'm now two... three? years post Adobe and Foxit and I still perk up when I see a blog post about PDFs. For today, I'm sharing a post that discusses [interactivity with SVGs and PDFs](https://vexlio.com/blog/svgs-and-pdfs-can-both-be-interactive/). The SVG part is pretty much what you expect - DOM stuff and JavaScript. The PDF part is a bit more esoteric as it [references](https://opensource.adobe.com/dc-acrobat-sdk-docs/library/jsapiref/index.html) a JavaScript engine built into PDFs from like the Mesozoic era of computers. 

## Animating a Dialog

The [Dialog](https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Elements/dialog) element still feels relatively new, but has great support (even in Safari) so there's no reason not to use it. The [View Transition API](https://developer.mozilla.org/en-US/docs/Web/API/View_Transition_API) also has stellar support. So this [article](https://pqina.nl/blog/animating-the-dialog-element-using-view-transitions/) by [pqina](https://pqina.nl/) is a great example of combining the two. I've used Dialog once or twice and want to use it more. The View Transition API still hasn't quite "clicked" with me yet but I'd like to use it more (where appropriate).
